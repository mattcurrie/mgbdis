# ポケットモンスター 緑 — シリアル通信プロトコル解析

ポケットモンスター 緑 (DMG-APAJ) のリンクケーブル通信プロトコルの技術文書。
対象アドレス: Bank 0 ($0BA7-$0D9A), Bank 1 ($51B3, $7388)。

## Game Boy シリアル通信の基礎

Game Boy のシリアルポートは SPI (Serial Peripheral Interface) ベースの同期式通信。

- **rSB** ($FF01): シリアルデータレジスタ (送受信兼用、8bit シフトレジスタ)
- **rSC** ($FF02): シリアル制御レジスタ
  - bit 7: 転送開始 (1=開始)
  - bit 0: クロック選択 (0=外部/スレーブ, 1=内部/マスター)
  - `$80` = 外部クロック受信待ち (スレーブ)
  - `$81` = 内部クロック転送開始 (マスター)

1バイト転送が完了すると **シリアル割り込み** ($0058) が発火。rSB には受信バイトが格納され、送信バイトは相手に送られる (全二重)。

### マスター / スレーブ

- **マスター**: 内部クロック (8192Hz) を生成。rSC=$81 で転送を開始する側
- **スレーブ**: 外部クロックを待つ。rSC=$80 で待機し、マスターの転送に応答

## HRAM 変数表

| アドレス | ラベル | 用途 |
|---------|--------|------|
| `$FF:$A9` | SerialTransferDone | 転送完了フラグ (0=待機中, 1=完了) |
| `$FF:$AA` | SerialRole | ロール状態 ($FF=未確定, $01=スレーブ, $02=マスター) |
| `$FF:$AB` | SerialSyncMode | 同期モード (1=初回同期, 0=通常転送) |
| `$FF:$AC` | SerialSendBuf | 送信バイト (次の転送で rSB にセットされる) |
| `$FF:$AD` | SerialRecvBuf | 受信バイト (ISR が rSB から読み取った値) |

## WRAM 変数表

| アドレス | 用途 |
|---------|------|
| `$CC3D` | ニブル受信バッファ (生データ) |
| `$CC3E` | ニブル受信バッファ (下位4bit抽出済み) |
| `$CC42` | ニブル送信バッファ |
| `$CC47` | タイムアウトカウンタ 上位バイト |
| `$CC48` | タイムアウトカウンタ 下位バイト |
| `$CCE0` | 通信コマンドバイト (ハンドシェイク=$10 等) |
| `$D051` | フレームカウンタ 下位 (マスタータイムアウト用) |
| `$D052` | フレームカウンタ 上位 (マスタータイムアウト用、初期値=$50) |
| `$D0F0` | 通信モード ($05=バトル) |
| `$CC34` | リンクモードフラグ (1=通信中) |

## SerialISR フロー ($0BA7)

```text
シリアル割り込み ($0058)
  │
  └→ SerialISR ($0BA7)
       │
       ├─ $AA = $FF (未確定)?
       │    └→ SerialISR_HandleRole ($0BC6)
       │         ├─ 受信値 = $02 → 自分はマスター
       │         │    └─ rSB = 0
       │         └─ 受信値 ≠ $02 → 自分はスレーブ
       │              └─ DIV遅延 → rSC = $80 (外部クロック待ち)
       │
       ├─ $AA = $01 (スレーブ)
       │    └─ rSB ← $AD, $AC → rSB, rSC = $80 (次の受信待ち)
       │
       └─ $AA = $02 (マスター)
            └─ rSB ← $AD, $AC → rSB (転送開始は呼び出し側)

  最後: $A9 = 1 (完了フラグ), $AC = $FE (アイドル)
```

## ハンドシェイクプロトコル ($7388)

両方の Game Boy が同時に接続を試み、先に相手のバイトを受信した側がスレーブになる。

```text
  GB-A                              GB-B
   │                                 │
   ├─ $AA = $FF (未確定)             ├─ $AA = $FF (未確定)
   │                                 │
   ├─ rSB=$02, rSC=$80              ├─ rSB=$02, rSC=$80
   │  (外部クロック待ち)              │  (外部クロック待ち)
   │                                 │
   ├─ rSB=$01, rSC=$81 ────────────→│
   │  (内部クロック送信)              ├─ ISR: 受信=$01
   │                                 │  → $AA=$01 (スレーブ確定)
   │                                 │
   │←──────────────── rSB=$02        │
   ├─ ISR: 受信=$02                  │
   │  → $AA=$02 (マスター確定)        │
   │                                 │
   ├─ 接続成功                       ├─ 接続成功
   │                                 │
  [同期信号 (SerialSendZero × 2)]
```

リトライ上限: 90回 ($5A)。超過で「つうしん エラー」表示。

## データ転送プロトコル (SerialExchangeBlock, $0BF1)

```text
呼び出し: HL=送信元, DE=受信先, BC=バイト数

  ループ (BCバイト分):
    1. [HL] → $AC (送信バイトセット)
    2. SerialSendAndWait 呼び出し
       ├─ マスター: rSC=$81 で転送開始
       └─ スレーブ: ISR完了待ち (タイムアウト付き)
    3. A = 受信バイト ($AD)
    4. 48サイクル遅延 (クロック安定化)
    5. 初回同期モード ($AB=1):
       ├─ $FD 受信 → HL-- してリトライ (同期ずれ回復)
       └─ それ以外 → $AB=0 (通常モードに移行)
    6. 通常モード ($AB=0):
       └─ A → [DE], DE++, BC--
```

特殊バイト:
- `$FE`: アイドル / 未準備 (リトライ対象)
- `$FD`: 同期ずれ信号 (初回モードでリトライ)

## トレード転送の3段階 ($51B3)

TradeExchangeData は SerialExchangeBlock を3回呼び出してパーティ情報を交換する。

| 段階 | 送信元 | 受信先 | サイズ | 内容 |
|------|--------|--------|--------|------|
| 1 | `$D106` | `$CD7C` | 17バイト ($11) | トレーナー名 |
| 2 | `$D117` | `$D812` | 359バイト ($167) | パーティデータ (ポケモン6体分) |
| 3 | `$C508` | `$C5D0` | 200バイト ($C8) | 種族ID・ニックネーム等 |

転送中は `rIE=$08` (シリアル割り込みのみ) に設定し、VBlank 等の割り込みを抑制。転送完了後に `rIE=$0D` に復帰。

マスターは転送前に `$CCE0` のコマンドバイトを2回送信して同期を取る。

## ニブル通信 ($0D57 / $0D6B)

メニュー選択の同期に使用される軽量プロトコル。1バイトの上位/下位4bitを分離して利用する。

```text
送信フォーマット: $6X (上位4bit = $6 がマーカー、X = データ)

SerialSendNibble ($0D57):
  1. SerialReceiveNibble で受信チェック
  2. $CC42 (送信値) + $60 → $AC
  3. マスターなら rSC=$81 で送信

SerialReceiveNibble ($0D6B):
  1. $AD の上位4bit = $60 か?
     ├─ No → return (無効データ)
     └─ Yes → 下位4bit → $CC3E, $AD=0 (処理済み)
```

SerialSyncWait ($0C66) がこのニブル通信をループで呼び出し、`$CC3E ≠ $FF` になるまで待機する。

## タイムアウト処理

2つのタイムアウト機構がある:

### 汎用タイムアウト ($CC47-$CC48)

16bit カウンタ。SerialCheckTimeout ($0D47) で `$CC47 | $CC48` が 0 かをチェック。

- ハンドシェイクでは $CC47=$5A (90回) で初期化
- スレーブの SerialSendAndWait では毎ループでインクリメント
- $FE (アイドル) 受信時にデクリメント → 0 でタイムアウト

### フレームカウンタ ($D051-$D052)

rIE=$08 (シリアルのみモード) 時にのみ動作。SerialSendAndWait 内で:

- $D051 をデクリメント → 0 で $D052 をデクリメント
- $D052 も 0 になると転送完了扱い
- 完了後は $D052=$50 にリセット
